# Структура базы данных телефонного справочника

## Таблица `contact`

| Поле | Тип | Описание | Обязательное |
|------|-----|----------|--------------|
| id | INTEGER | Уникальный идентификатор контакта | Да |
| full_name | VARCHAR(100) | ФИО | Да |
| work_phone | VARCHAR(20) | Рабочий телефон | Нет |
| mobile_phone | VARCHAR(20) | Сотовый телефон | Нет |
| email | VARCHAR(100) | Электронная почта | Нет |
| notes | TEXT | Примечания | Нет |

## Таблица `admin_user`

| Поле | Тип | Описание | Обязательное |
|------|-----|----------|--------------|
| id | INTEGER | Уникальный идентификатор администратора | Да |
| username | VARCHAR(80) | Имя пользователя | Да |
| password_hash | VARCHAR(120) | Хеш пароля | Да |

## Описание полей

### Таблица `contact`
- `id`: Автоматически генерируемый первичный ключ
- `full_name`: Полное имя человека (ФИО), обязательное поле
- `work_phone`: Рабочий телефон, может быть пустым
- `mobile_phone`: Сотовый телефон, может быть пустым
- `email`: Адрес электронной почты, может быть пустым
- `notes`: Дополнительная информация о контакте, может быть пустым

### Таблица `admin_user`
- `id`: Автоматически генерируемый первичный ключ
- `username`: Уникальное имя пользователя для входа в систему
- `password_hash`: Хешированный пароль (не хранится в открытом виде)

## Связи между таблицами

Между таблицами нет прямых внешних ключей, так как каждый контакт не привязан к конкретному администратору. Администраторы могут управлять всеми контактами.

## Индексы

В текущей реализации индексы не определены, но при необходимости могут быть добавлены для полей:
- `full_name` для ускорения поиска по ФИО
- `email` для ускорения поиска по электронной почте
- `username` для ускорения аутентификации
